# QR Guard Browser

QR 코드 사이트를 **로컬 잠금 뒤에 숨겨** 주는 데스크톱 브라우저입니다. 지정한 사이트를 실제 Chromium 화면으로 띄우되, **지역별 인증코드로 잠금을 해제한 동안만** QR을 보여주고, 설정한 시간이 지나거나 QR 화면이 감지되면 **자동으로 다시 가립니다.** (macOS · Windows)

## 💿 설치

> ⚠️ 코드 서명이 되어 있지 않아 **첫 실행 때만** OS 보안 경고가 뜹니다. 아래 절차대로 한 번만 허용하면 됩니다.

### Windows
1. `QRGuardBrowserSetup.exe`를 받아 **더블클릭**합니다.
2. **"Windows의 PC를 보호했습니다"** 파란 창이 뜨면 → **`추가 정보`** 클릭 → **`실행`** 클릭.
3. 별도의 설치 마법사 없이 **자동으로 설치된 뒤 바로 앱이 열립니다** (Slack·Discord·VS Code와 같은 방식). 시작 메뉴에 바로가기가 생기고, 제거는 *제어판 → 프로그램 추가/제거*에서 할 수 있습니다.

### macOS
1. `QR Guard Browser.dmg`를 받아 더블클릭으로 열고, **앱 아이콘을 `Applications` 폴더로 드래그**합니다.
2. 첫 실행 시 *"확인되지 않은 개발자"* 경고가 나오면, 앱을 **마우스 우클릭(또는 Control+클릭) → `열기`** → 다시 **`열기`**.
3. 한 번 허용하면 이후에는 더블클릭으로 바로 열립니다.

## 🚀 사용 방법

### 1. 첫 실행 설정
처음 켜면 기본 정보를 한 번 설정합니다.

![첫 실행 설정 화면](docs/images/01-setup.png)

1. **QR 사이트 주소** — 띄울 사이트의 주소(보통 로그인 페이지). 시작 주소일 뿐, 로그인 후 다른 주소로 이동해도 됩니다.
2. **관리자 코드** — 설정 진입과 사이트 로그인 모드에 쓰는 관리자 비밀번호.
3. **지역** — 잠금을 해제할 지역(부서/지점) 이름.
4. **인증 코드** — 해당 지역의 잠금 해제 코드.
5. **초기 설정 저장** — `지역 추가`로 여러 지역을 등록할 수 있고, *노출 시간*(QR을 보여줄 초)과 *유휴 자동잠금*(사용이 없을 때 잠기는 초)도 함께 정합니다.

### 2. 잠금 해제 (QR 보기)
평소에는 잠겨 있고, QR을 보려면 지역 인증을 합니다.

![잠금 화면](docs/images/02-locked.png)

1. **지역** — 설정에 등록한 지역 목록에서 **드롭다운으로 선택**합니다.
2. **인증 코드** — 그 지역의 인증 코드를 입력합니다.
3. **잠금 해제** — QR이 *노출 시간* 동안만 표시된 뒤 자동으로 다시 잠깁니다.
4. **사이트 로그인** — 로그인을 여러 단계 거쳐야 QR이 나오는 사이트용 모드(아래 4번 참고).
5. **설정** — 관리자 코드를 입력해 설정 화면으로 들어갑니다.

### 3. 브라우저 도구 (잠금 해제 중 상단 막대)
QR이 보이는 동안에는 위쪽에 일반 브라우저처럼 도구 막대가 나타납니다.

![상단 도구 막대](docs/images/03-toolbar.png)

1. **뒤로** · 2. **앞으로** · 3. **새로고침** — 일반 브라우저와 동일하게 페이지를 이동/갱신합니다.
4. **주소창** — 현재 주소가 표시되고, 직접 입력해 다른 페이지로 이동할 수 있습니다.
5. **지금 잠그기** — 노출 시간이 남아 있어도 즉시 잠급니다. (왼쪽의 카운트다운이 남은 노출 시간입니다.)

> 소셜·OAuth 팝업 로그인, 생체인증(WebAuthn) 등도 동작합니다. 단, 로그인 팝업도 잠금 안쪽에서 열려 **QR이 새어 나오지 않습니다.**

### 4. 사이트 로그인 모드 (다단계 로그인)
로그인 → 메뉴 이동 → 여러 번 클릭해야 QR이 나오는 사이트를 위한 모드입니다.

![사이트 로그인 다이얼로그](docs/images/04-sitelogin.png)

1. 잠금 화면의 **사이트 로그인**을 누르면 **관리자 코드**를 묻습니다.
2. 인증하면 로그인/탐색을 자유롭게 할 수 있고, **QR 화면(설정의 "QR 화면 제목"과 일치하는 페이지)이 감지되는 순간 자동으로 잠깁니다.**

### 5. 설정
관리자 코드로 진입해 동작을 바꿀 수 있습니다.

![설정 화면](docs/images/05-settings.png)

- **QR 사이트 주소 / 노출 시간 / 유휴 자동잠금** — 기본 동작 값.
- **QR 화면 제목** — 이 문구가 페이지 제목에 들어가면 QR로 인식해 **자동 잠금**합니다(막거나 제한하는 용도가 아니라 "언제 가릴지" 신호).
- **지역 관리** — 지역 추가·이름 변경·인증 코드 재설정·삭제.
- 이 밖에 **관리자 코드 변경**, **저장된 로그인**(비밀번호 자동저장/자동완성 관리), **인증 기록(감사 로그) 내보내기**, **QR 세션 초기화**(쿠키/캐시 삭제)가 있습니다.

## 🔒 동작 원리 (요약)
- QR 화면은 **지역/관리자 인증을 통과한 동안에만** 표시되고, 그 외에는 항상 가려집니다.
- 인증 코드와 관리자 코드는 **scrypt 해시 + OS 키체인(safeStorage)**로 봉인되어 저장되며, 평문으로 보관하지 않습니다.
- QR 사이트 비밀번호 자동저장도 OS 키체인으로 암호화됩니다.

```bash
npm ci

npm test           # 단위 테스트
npm run test:e2e   # E2E (Playwright)
npm run make       # 설치본(.dmg/.exe) 빌드
```

릴리스(태그 `v*` 푸시 → main)는 GitHub Actions가 macOS·Windows 양쪽 설치본을 자동 빌드합니다.
